Compliance Language Audit — Frontend

Goal: find any wording that reads like gambling / betting / casino / prediction-market (Polymarket, Kalshi) and propose neutral replacements. Audit only — no code changed. Scanned all of src/ (2026-06-01).

Summary

  • GOOD The words bet / betting / gambling / wager appear only on the Disclaimer page, used correctly in the negative ("The Analyst is not a gambling, betting… service"). That is protective legal language — keep it.
  • GOOD No casino, jackpot, bookie, lottery, Polymarket, Kalshi, cash-out, deposit/withdraw real money anywhere in user-facing copy. The platform already deliberately avoids the phrase "prediction market".
  • FLAG The only real flags are the word "odds" (3 places) and "payout" (2 places). Simple text swaps, no logic involved.
